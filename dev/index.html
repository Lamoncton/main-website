Back-end

  • Node JS version when writing this: v19.4.0

File system

Each folder with index.js is a valid URL to connect to from root. In each folder, it is possible to provide with custom style.css to overwrite the default /style.css.

When creating a new file that you wish to show up in the Release, you must explcitly add it to build/build.js

When creating a new file, it should be named with singular noun. It is script.js, not scripts.js. Folders and URL folders are not subject to this convention.

Translations

Translation requires you to build the website with the build script.

In each folder, there will be an index.html files to dictate the layout and the translation keys. In each folder, there will be .json files with language code.

Translation keys will be replaced with translated content in that perspective language, then put into the correct folder according to the key URL. Translation key should be in English.

Because the replace is pure plain text, it is recommened to put translation key inside >translationKey⪪ for keys with confusable name. The best practice is to name keys with long enough snake_case name. Snake case is to prevent conflict with CSS camel case.

A certain control key will be ignored (like URL)

transContent

Build script

The build script is important in a sense that it will put all html-src into the element itself, removing the need to execute script to fetch, hence help with SEO because the meta tags are present.

Development

To build a complete website, run node build/build.js (without the release flag). The script will build everything to build/Release. Noted that with this kind of dev environment, you need to build everytime you make an edit.

Due to CORS, you must run a static server to fully see the page. Python3's simple HTTP server was known to have problem with encoding, so it is recommended to use Node JS static server with npx http-server -o ./

Deployment and Release

When wishing to release, run build/build.js. The script will simply copy pre-specified files and folders to build/Release (ignored folder in .GITIGNORE). The script will then apply translation. After that, the script will switch to Release branch, then move the files back to the main folder. Finally, it will auto-commit and push to the cloud, before returning back to the main branch.

PLEASE DO NOT WORK ON RELEASE BRANCH!
